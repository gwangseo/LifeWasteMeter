# 현재 알고리즘 분석

## 1. 앱 실행 구분 방법
- **현재 방법**: `TYPE_WINDOW_STATE_CHANGED` 이벤트의 `packageName`으로 구분
- **문제점**: 없음 (정상 작동)

## 2. 스크롤 vs 클릭 구분 방법
- **현재 방법**: 
  - `TYPE_VIEW_SCROLLED` 이벤트만 사용
  - `event.scrollY` 값의 변화량으로 판단
- **문제점**: 
  - 많은 앱에서 `TYPE_VIEW_SCROLLED` 이벤트가 발생하지 않음
  - `scrollY` 값이 항상 정확하지 않음
  - 클릭과 스크롤을 구분할 수 없음

## 3. 쇼츠/릴스 구분 방법
- **현재 방법**: 구분하지 않음
- **문제점**: 
  - 쇼츠/릴스는 수직 스크롤이 아니라 수평 스와이프
  - 일반 피드와 구분 불가능

## 개선 방안

### 제안 1: 노드 정보 활용
- 스크롤 가능한 뷰를 찾아서 그 뷰의 실제 변경을 감지
- 노드의 bounds나 content description으로 구분

### 제안 2: 제스처 패턴 분석
- 제스처의 방향과 거리를 분석하여 스크롤인지 클릭인지 구분
- 수평 스와이프(쇼츠/릴스)와 수직 스크롤(피드) 구분

### 제안 3: 하이브리드 접근
- 여러 이벤트 타입을 조합하여 사용
- 앱별로 다른 전략 적용
